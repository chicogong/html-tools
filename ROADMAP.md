# WebUtils 2.0 迭代演进路线图 (Roadmap)

本文档记录了 WebUtils 在完成 SSG (静态站点生成) 架构升级后的后续重点迭代方向。

## 🎯 优先级 1：极致的本地开发体验 (DX Tooling)

**状态**: 待处理 (Todo)
**描述**:
引入本地开发热重载（Hot Reload）机制。
目前开发者在本地修改 HTML 片段后，需要手动运行 `npm run build` 才能看到最终效果。
**计划**:
编写一个轻量级的 `scripts/dev.js`。利用 `browser-sync` 或原生的 `fs.watch` 监听 `tools/**/*.html` 和 `src/layout.ejs` 的变化。只要文件一保存，自动增量触发 `build` 并在浏览器中实现无刷新热重载。
**价值**: 让贡献者的开发体验比肩 Vue/React 等现代化框架。

## 🎯 优先级 2：PWA 离线桌面级应用 (Offline-First)

**状态**: 待处理 (Todo)
**描述**:
将整个工具箱升级为 PWA（渐进式 Web 应用）。
**计划**:
既然所有构建产物现在都收拢在了 `dist/` 文件夹中，我们可以在构建流程的最后加入 Workbox (或原生 Service Worker) 脚本。它会自动抓取 `dist/` 目录下的所有文件并生成缓存清单（Manifest）。
**价值**: 用户只需在浏览器点击“安装”，这 1000 多个工具就可以变成原生的桌面软件或手机 App，在完全断网的环境下零延迟运行，完美契合本项目的核心理念。

## 🎯 优先级 3：零成本多语言国际化 (i18n Architecture)

**状态**: 待处理 (Todo)
**描述**:
利用 SSG 引擎的红利，批量生成多语言版本的静态网站。
**计划**:
升级 `scripts/build-ssg.js`，让它在构建时循环读取 `i18n/en.json` 和 `i18n/zh-CN.json`。
除了在根目录生成中文站，还会额外生成一份 `/dist/en/` 英文站。模板引擎会自动替换掉侧边栏、介绍、以及分类文字。
**价值**: 真正实现维护一份工具 HTML 代码，免费生成全球多国语言版本，极大地扩展 WebUtils 的海外受众。
