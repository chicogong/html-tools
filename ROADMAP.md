# WebUtils 2.0 迭代演进路线图 (Roadmap)

本文档记录了 WebUtils 在完成“零构建架构 (Zero-Build Architecture)” 升级后的后续重点迭代方向。

## 🎯 已完成：极致的本地开发体验 (DX Tooling) 与零构建架构

- 所有工具已经全部迁移至单文件独立 HTML 格式。
- 修改工具不再需要运行任何构建脚本，只需保存并在浏览器中刷新（F5）即可预览，真正达到了所见即所得。

## 🎯 已完成：PWA 离线桌面级应用 (Offline-First)

- 已配置 `sw.js` 和 `manifest.json`，实现了断网可访问的离线缓存。
- 用户可以直接将工具集作为桌面应用或手机 App 安装到本地使用。

## 🎯 优先级 1：独立 HTML 工具生态 (Standalone HTML Tool Ecosystem)

**状态**: 进行中 (In Progress)
**描述**:
架构调整已经结束，现在的重点不是继续单纯堆工具数量，而是把每个工具打磨成可在线访问、可分享、可下载到本地、可被社区共建的独立 HTML 小应用。
**计划**:

1. 建立新工具质量基线：SEO、结构化数据、共享基座、复制反馈、移动端、隐私说明。
2. 设计 standalone 导出能力，让单个工具可打包成一个可下载、可 `file://` 打开的 HTML 文件。
3. 建立共建模板或 `create:tool` 脚手架，让贡献者用一个 HTML 文件和一条 `tools.json` 记录即可贡献工具。
4. 梳理现有高价值工具，优先打磨体验和可信度，而不是继续追求工具数量。

详见 `docs/product-direction.md`。

## 🎯 已完成：性能优化与 Lighthouse 跑分提升

**状态**: ✅ 已完成 (Done)
**描述**:
进一步提高页面加载速度，保证 Google Lighthouse 性能跑分处于满分状态。
**计划**:
已完成 `INITIAL_LOAD` 分批渲染优化，消除了首屏长任务阻塞。为复杂 CSS 滤镜动画（如极光生成器）增加了 GPU 硬件加速，彻底解决卡顿问题。

## 🎯 （可选）国际化 (i18n)

**状态**: 搁置 (On Hold)
**描述**: 目前暂不考虑推进多语言版本，后续如果有强烈的海外扩张需求，可引入前端无编译的 JSON 字典替换方案。
