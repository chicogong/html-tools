# WebUtils 2.0 迭代演进路线图 (Roadmap)

本文档记录了 WebUtils 在完成“零构建架构 (Zero-Build Architecture)” 升级后的后续重点迭代方向。

## 🎯 已完成：极致的本地开发体验 (DX Tooling) 与零构建架构
- 所有工具已经全部迁移至单文件独立 HTML 格式。
- 修改工具不再需要运行任何构建脚本，只需保存并在浏览器中刷新（F5）即可预览，真正达到了所见即所得。

## 🎯 已完成：PWA 离线桌面级应用 (Offline-First)
- 已配置 `sw.js` 和 `manifest.json`，实现了断网可访问的离线缓存。
- 用户可以直接将工具集作为桌面应用或手机 App 安装到本地使用。

## 🎯 优先级 1：工具质量与功能拓展 (Tool Expansion)
**状态**: 进行中 (In Progress)
**描述**:
架构调整已经结束，现在的重点是丰富工具内容并提升单个工具的质量。
**计划**:
1. 梳理现有的 1000+ 工具，优化交互不佳或 UI 老旧的工具。
2. 根据用户反馈或热门需求，新增高质量的实用工具（如音视频处理、AI 效率辅助等）。
3. 确保所有新加的工具都严格遵循“零构建、单文件”的原则。

## 🎯 优先级 2：性能优化与 Lighthouse 跑分提升
**状态**: 待处理 (Todo)
**描述**:
进一步提高页面加载速度，保证 Google Lighthouse 性能跑分处于满分状态。
**计划**:
检查资源加载情况，延迟加载非关键 JS/CSS，压缩关键 SVG 与图片资源，移除冗余的 CSS 选择器。

## 🎯 （可选）国际化 (i18n)
**状态**: 搁置 (On Hold)
**描述**: 目前暂不考虑推进多语言版本，后续如果有强烈的海外扩张需求，可引入前端无编译的 JSON 字典替换方案。
